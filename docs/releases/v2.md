# Sandra V2

## Objective

Turn Sandra from a basic public EdLight assistant into a more accurate,
tool-routed, knowledge-grounded gateway for the EdLight ecosystem.

Sandra V2 focuses on answer accuracy, tool routing, retrieval quality,
and stable streaming tool execution.

## Current Status

**✅ Sandra V2 is signed off and shipped — March 25, 2026.**

All validation gates passed, benchmark prompts reviewed in live chat,
operator QA completed in admin dashboard. Ready for production deployment.

## Included in V2

- Improved tool routing for EdLight platforms
- Accurate course responses for EdLight Academy and EdLight Code
- Better repository retrieval grounding
- Improved session/context continuity
- Stable streaming tool-call execution
- Knowledge-source transparency
- Evaluation tests for core EdLight prompts

## Platforms Covered

- EdLight Academy
- EdLight Code
- EdLight News
- EdLight Initiative
- EdLight Nexus
- EdLight Labs
- ESLP (EdLight Summer Leadership Program)

## Excluded from V2

The following roadmap items are **not part of V2**:

- WhatsApp integration
- Instagram integration
- Email integration
- Voice interface
- Autonomous agent actions
- Complex user authentication flows

These belong to later roadmap versions.

## Success Criteria

Sandra V2 is complete when:

- Sandra answers EdLight platform questions accurately
- Tool selection is correct for benchmark prompts
- Repository content is used to ground answers
- Streaming chat does not break tool-call continuity
- Operator QA passes for admin auth, repo listing, single-repo indexing, index-all, and partial failures
- All benchmark prompts pass with grounded or explicitly unavailable answers
- All tests pass
- `tsc --noEmit` passes
- Build succeeds

## Release Gates

- Canonical contract review completed for `/api/chat`, `/api/chat/stream`, `/api/repos`, `/api/index`, and `/api/health`
- Streaming `done` terminal payload is documented and verified
- Operator QA checklist completed
- Benchmark/eval checklist completed
- Legacy docs updated where behavior has changed

See `docs/releases/v2_signoff.md` for the active checklist.

## What Comes Next

After V2 signoff, the next planned phase is:

- memory and continuity hardening
- production-ready knowledge/indexing operations
- authenticated user foundations

WhatsApp, Instagram, email, voice, and broader agentic workflows remain later roadmap work.
